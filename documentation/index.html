Installation

Follow the steps below to setup your site template:

  • Unzip the downloaded package and open the /reala-main folder to find frontend & backend files
  • Frontend Setup:

    Open reala-frontend folder in your terminal and follow the steps given below

    • Run npm install in terminal to install all the uses packages.
    • In case you don't have Node.js & NPM package manager on your machine, here how to add it:
      • Download Node.js from browser. This will automatically add npm with it.
      • Open Command Prompt and check node and npm versions to make sure they have been added.
    After installing packages its time to setup your .env file with your api and necessary links. [NOTE: you will find your api after setting the backend file]
  • NEXT_PUBLIC_API_URL is for api url. Replace your api link with API_URL
  • NEXT_PUBLIC_MAILCHAIMP_LINK is for mailchimp url. Replace your api link with MAILCHAIMP_LINK
  • NEXT_PUBLIC_CONTACTFROM is for fromspree link. Replace your api link with FROMSPREE.
    • here is a sample you do that.

Now you're ready to start the project. simply run npm run dev to run the project

  • Backend Setup:

    Open reala-backend folder in your terminal and follow the steps given below

    • Run npm install in terminal to install all the uses packages.
    • Now you need to set .env
    • To collect .env data you have to create a stapi project. There's no others way :((
    • Create a separate file by running npx create-strapi-app@latest my-project & grave the .env form there and paste it in reala-backend file.

      • Now you're ready to start the project. simply run npm run develop to run the project

    • Once you run the project you need to register on strapi and change some settings.

    • To do so go to Settings > Roles > Authenticated
    • Click on Blog and checkmark them like this photo
    • Click on Property and checkmark them like this photo
    • Click on Category and checkmark them like this photo
    • Click on Upload and checkmark them like this photo
    • Click on Users-permissions and checkmark them like this photo

    You need to do exact same Settings in Public section too. For that go to Settings > Roles > Public and change exact same things you did in Authenticated

  • Development server:
    • Now add your content in Strapi and deploy it to get a api
    • There are many server site deploy platform exist on internet. However, we suggest amazon lightsail to do so. We have found this very easy to navigate and use.

Template Customization

And to change styles you need to go to sass folder inside src/assets

Process of customize style with SCSS ðŸ‘‡

We have added SASS .scss files in template. If you know how to use SASS you can change sass files.

Open the assets/sass/base/_base.scss, assets/sass/base/_typography.scss and Edit the values according to your needs. If you need more Advanced Setup then you can Edit the Respective Files yourself which have been branched inside the same Folder. It is completely at your discretion only to include the Required .scss Files you need to minimize the amount of CSS & including only the Styles of the Blocks you need. This can be setup in your main.scss File.

All colors of template operating from _variable.scss file. Open sass/abstracts/_variable.scss to change colors as you want.

To customize responsive layout you have to open sass/abstracts/mixins.scss. There you can change media queries value as you want.

And All components based scss are in sass/components. Find the page you want to change css there.

Typography

Font Family

We used two different font-family. One for Heading and title and another one for rest. You can simply use another font-family from Google Font by importing it inside assets/sass/base/_typography.scss. 



  <!-- ========== Fonts ========== -->
  <!--====== Heading font ======-->
  @import url("https://fonts.googleapis.com/css2?family=Raleway:ital,wght@0,400;0,700;1,400;1,700&display=swap")
  <!--====== Body font ======-->
  @import url("https://fonts.googleapis.com/css2?family=Poppins:ital,wght@0,400;0,700;1,400;1,700&display=swap")
  <!-- ========== Add your fonts here or you can replace these two ========== -->

  <!--====== Font Family ======-->
  $font-heading: "Raleway", sans-serif;
  $font-display: "poppins", sans-serif;

After importing fonts inside _typography.scss of index file you just need to add your font family variable in same file. Set your chosen family to $font-heading or $font-display as their value just like the example given up
